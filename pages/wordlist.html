Creating a Wordlist Using Crunch from a Git Repository
Wordlists are essential for penetration testing tasks like brute-forcing passwords or directories. Crunch is a powerful wordlist generator available on Kali Linux, which can be installed from a Git repository if not pre-installed. This guide covers cloning the Crunch repository and generating a wordlist. Always use wordlists ethically and with explicit permission in a controlled lab environment to avoid legal issues.
Steps to Create a Wordlist Using Crunch on Kali Linux
1. Verify Crunch Installation
Check if Crunch is installed:
crunch -v
Explanation: The -v flag displays the version of Crunch, confirming whether it’s installed and ready for use.
2. Install Crunch via Git Repository
If Crunch is not installed, clone it from its Git repository and install it:
git clone https://github.com/crunchsec/crunch.git
cd crunch
make
sudo make install
Explanation:
- git clone https://github.com/crunchsec/crunch.git: Downloads the Crunch source code from its GitHub repository.
- cd crunch: Navigates into the cloned directory.
- make: Compiles the Crunch source code.
- sudo make install: Installs Crunch system-wide, making it accessible from the command line.
3. Get Authorization
Obtain explicit written permission before using wordlists for testing on any system. For practice, use a lab environment like Metasploitable 2 or a personal test server.
4. Generate a Basic Wordlist
Create a wordlist with a specific length and character set (e.g., 4-character passwords using lowercase letters):
crunch 4 4 -f /usr/share/crunch/charset.lst lalpha -o wordlist.txt
Explanation:
- crunch 4 4: Specifies minimum and maximum word length (both 4 characters).
- -f /usr/share/crunch/charset.lst lalpha: Uses the lowercase alphabet character set from Crunch’s charset file.
- -o wordlist.txt: Saves the generated wordlist to a file named wordlist.txt.
5. Generate a Custom Wordlist
Create a wordlist with specific patterns (e.g., 6-character passwords with a mix of letters and numbers, starting with "pass"):
crunch 6 6 -t pass@@ -o custom_wordlist.txt
Explanation:
- crunch 6 6: Sets the word length to exactly 6 characters.
- -t pass@@: Defines a pattern where "pass" is fixed, and @@ represents two placeholder characters (defaulting to alphanumeric).
- -o custom_wordlist.txt: Saves the output to custom_wordlist.txt.
6. Optimize Wordlist Size
Compress the wordlist to reduce storage (optional):
gzip wordlist.txt
Explanation: The gzip command compresses the wordlist file to save disk space, useful for large wordlists.
7. Verify Wordlist
Check the wordlist content:
head -n 10 wordlist.txt
Explanation: Displays the first 10 lines of the wordlist to verify its contents and ensure it meets the desired format.
Example output:
aaaa
aaab
aaac
aaad
aaae
aaaf
aaag
aaah
aaai
aaaj
Explanation: Shows a sample of generated words, confirming the wordlist follows the specified character set and length.
8. Use Wordlist (Optional)
Use the wordlist with a tool like Hydra for testing (in a lab environment):
hydra -l admin -P wordlist.txt [Metasploitable_IP] http-post-form "/login.php:username=^USER^&password=^PASS^:Invalid"
Explanation:
- hydra: A password-cracking tool.
- -l admin: Specifies the username to test (e.g., "admin").
- -P wordlist.txt: Uses the generated wordlist for passwords.
- [Metasploitable_IP] http-post-form: Targets a login page on the Metasploitable 2 server.
- /login.php:username=^USER^&password=^PASS^:Invalid: Defines the login form parameters and failure message.
Important Notes
- Use Crunch and wordlists ethically; unauthorized brute-forcing is illegal.
- Test in a controlled lab environment (e.g., Metasploitable 2) to practice safely.
- Large wordlists can consume significant disk space; plan storage accordingly.
- Comply with all legal and organizational policies.
