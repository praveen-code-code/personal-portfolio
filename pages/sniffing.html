Packet Sniffing with Wireshark and Metasploitable 2
Wireshark is a free, open-source packet analyzer used for network troubleshooting, analysis, and education. It captures and inspects network traffic in real-time. Metasploitable 2 is an intentionally vulnerable virtual machine (VM) designed for testing security tools like Metasploit. Here, we'll use Wireshark to sniff traffic involving Metasploitable 2 on Kali Linux. Always obtain permission and ensure you're in a controlled lab environment to avoid legal issues, as unauthorized sniffing can violate privacy laws.
Steps to Perform Packet Sniffing Using Wireshark with Metasploitable 2 on Kali Linux
1. Set Up Metasploitable 2
Download and run Metasploitable 2 VM (e.g., via VirtualBox or VMware). Ensure it's on the same network as your Kali machine.
Note: Use a bridged or NAT network adapter for traffic visibility.
2. Verify Wireshark Installation
Wireshark is pre-installed on Kali Linux.
Check version:
wireshark --version
Explanation: The --version flag displays the current version of Wireshark, confirming it's installed and ready for use.
3. Start Wireshark
Launch Wireshark with root privileges (required for packet capture):
sudo wireshark
Explanation: sudo runs Wireshark with elevated permissions to access network interfaces for capturing raw packets. Without sudo, capture options may be limited.
4. Get Authorization
Obtain explicit written permission if sniffing involves real networks or systems. For lab use with Metasploitable 2, ensure it's isolated.
5. Select Capture Interface
In Wireshark GUI:
- Go to Capture > Options.
- Select the network interface (e.g., eth0) connected to the Metasploitable 2 network.
- Click "Start" to begin capturing.
Explanation: The interface is the network card through which traffic flows. Choosing the correct one (e.g., via ip link show in terminal) ensures you capture relevant packets.
6. Generate Traffic
Interact with Metasploitable 2 to generate network traffic:
- Ping the Metasploitable 2 IP:
ping [Metasploitable_IP]
Explanation: ping sends ICMP echo requests to the target IP, creating simple traffic that Wireshark can capture and display as request/reply packets.
- Or access a service (e.g., HTTP):
curl http://[Metasploitable_IP]
Explanation: curl fetches the web page from the target's HTTP server, generating TCP and HTTP packets for analysis.
7. Apply Filters
In Wireshark, use display filters to focus on traffic (e.g., from Metasploitable 2):
ip.addr == [Metasploitable_IP]
Explanation: This filter shows only packets where the source or destination IP matches the target's IP, narrowing down the capture to relevant traffic.
For HTTP traffic:
http
Explanation: Filters packets containing HTTP protocol data, useful for inspecting web requests and responses.
8. Stop Capture and Analyze
- In Wireshark: Capture > Stop.
- Inspect packets: Click on a packet to view details in the packet details pane.
Explanation: Stopping the capture freezes the data for analysis. You can expand protocol layers (e.g., Ethernet, IP, TCP) to see headers, payloads, and potential sensitive data like unencrypted credentials.
9. Review Results
Look for insights such as:
- Unencrypted data (e.g., HTTP credentials).
- Protocol vulnerabilities in Metasploitable 2 services.
Example captured packet summary:
No. Time Source Destination Protocol Info
1 0.000000 Kali_IP Metasploitable_IP ICMP Echo (ping) request
Explanation: This shows a ping request packet, including timestamp, IPs, protocol, and info. Analyze for patterns, anomalies, or security issues. Verify findings manually and document for reporting.
Important Notes
- Use Wireshark ethically; sniffing can reveal sensitive information.
- In production, use encrypted protocols (e.g., HTTPS) to prevent sniffing.
- Test in a lab environment with Metasploitable 2 to practice safely.
- Comply with all legal and organizational policies.
